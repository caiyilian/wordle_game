# 猜单词游戏 (Web版) 规则说明

这是一个基于 Web 的多人互动式 Wordle（猜单词）游戏。你可以在网页上通过聊天框发送指令来开始游戏、猜测单词或进行聊天。

### 1. 游戏目标
在有限的次数内猜出一个随机选中的英文单词。

### 2. 游戏指令 (在聊天框输入)

*   **开始游戏**：
    *   输入 **`猜单词`**：开始一个默认长度（5个字母）的新游戏。
    *   输入 **`猜单词 <数字>`**：开始指定长度的游戏（例如 `猜单词 3` 或 `猜单词 6`）。
    *   *注*：数字必须大于等于 3。
*   **结束游戏**：
    *   输入 **`结束游戏`**：强制结束当前正在进行的游戏。系统将判负，并公布正确答案及其中文释义。
*   **获取提示**：
    *   输入 **`提示`**：在游戏界面顶部显示当前已知的线索（即所有已猜出且位置正确的字母）。
    *   *注*：提示栏**不会自动更新**。如果你通过新的猜测获得了更多线索，需要再次发送 `提示` 指令来刷新顶部的显示。

### 3. 游戏玩法与判定

*   **猜测单词**：
    *   直接在输入框发送单词即可。
    *   **长度检查**：单词长度必须与当前题目长度一致。
    *   **合法性检查**：必须是合法的英文单词。
    *   **去重检查**：不能输入已经猜过的单词。
    *   *反馈*：如果输入符合上述条件，会被视为一次有效猜测，并在网格中显示颜色反馈。如果不符合（如拼写错误、重复），系统会在聊天框中给出提示，不扣减猜测次数。

*   **聊天模式**：
    *   任何不符合猜测规则（例如长度不对、含有数字/符号）或非特定指令的输入，都会被视为**普通聊天消息**显示在右侧记录中。
    *   游戏结束后（无论输赢），所有的输入都会变为聊天模式，直到开始新游戏。

### 4. 颜色反馈含义

*   🟩 **绿色**：字母包含在单词中，且**位置正确**。
*   🟨 **黄色**：字母包含在单词中，但**位置不正确**。
*   ⬜ **灰色**：字母**不包含**在单词中。

### 5. 游戏结算

*   **胜利**：在规定机会（单词长度 + 1 次）内猜出正确单词。
*   **失败**：机会用完仍未猜出，或手动发送“结束游戏”。
*   **结果**：游戏结束时，系统会自动显示正确单词及其中文释义。

### 6. 技术栈与启动

*   **后端**：Python (Flask)
*   **前端**：HTML5, CSS3, JavaScript
*   **启动方式**：在终端运行 `python app.py`，然后访问 `http://127.0.0.1:5000/`。
